# appleappstoreconnect_subscription_availability (Data Source)

Reads the territory availability of a subscription. The App Store Connect API does not currently expose a public endpoint to modify subscription availability, so this is provided as a read-only data source.

## Example Usage

```hcl
data "appleappstoreconnect_subscription_availability" "monthly" {
  subscription_id = appleappstoreconnect_subscription.monthly.id
}

output "available_territories" {
  value = data.appleappstoreconnect_subscription_availability.monthly.available_territories
}
```

## Schema

### Required

- `subscription_id` (String) The ID of the subscription whose availability is read.

### Read-Only

- `available_in_new_territories` (Boolean) Whether the subscription is automatically made available in territories Apple adds in the future.
- `available_territories` (Set of String) The set of territory codes where the subscription is available.
- `id` (String) The unique identifier of the availability configuration.
