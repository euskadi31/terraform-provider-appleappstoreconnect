# appleappstoreconnect_subscription_price (Resource)

Manages the price of a subscription in a single territory. Subscriptions are priced per territory (there is no single price schedule like In-App Purchases), so create one resource per territory. Price point IDs are resolved with the `appleappstoreconnect_subscription_price_point` data source. Changing any attribute replaces the price.

## Example Usage

```hcl
data "appleappstoreconnect_subscription_price_point" "usd_499" {
  subscription_id = appleappstoreconnect_subscription.monthly.id
  territory       = "USA"
  customer_price  = "4.99"
}

resource "appleappstoreconnect_subscription_price" "usd" {
  subscription_id             = appleappstoreconnect_subscription.monthly.id
  subscription_price_point_id = data.appleappstoreconnect_subscription_price_point.usd_499.id
  territory                   = "USA"
}
```

## Schema

### Required

- `subscription_id` (String) The ID of the subscription this price applies to. Changing this forces a new resource.
- `subscription_price_point_id` (String) The price point ID for this territory (resolve it with the `appleappstoreconnect_subscription_price_point` data source). Changing this forces a new resource.
- `territory` (String) The territory code this price applies to (e.g. `USA`). Changing this forces a new resource.

### Optional

- `preserve_current_price` (Boolean) When `true`, existing subscribers keep their current price and only new/renewing subscribers get this price. Changing this forces a new resource.
- `start_date` (String) The date the price takes effect (ISO 8601, e.g. `2025-06-01`). Omit for immediate effect. Changing this forces a new resource.

### Read-Only

- `id` (String) The unique identifier of the subscription price.

## Import

Subscription prices have no individual read endpoint, so import requires both the subscription ID and the price ID:

```bash
terraform import appleappstoreconnect_subscription_price.usd <subscription_id>:<price_id>
```
