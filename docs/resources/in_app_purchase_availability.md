# appleappstoreconnect_in_app_purchase_availability (Resource)

Manages the territory availability of an In-App Purchase. Changing any attribute replaces the availability configuration.

## Example Usage

```hcl
resource "appleappstoreconnect_in_app_purchase_availability" "premium" {
  in_app_purchase_id           = appleappstoreconnect_in_app_purchase.premium.id
  available_in_new_territories = true
  available_territories        = ["USA", "FRA", "GBR"]
}
```

## Schema

### Required

- `available_in_new_territories` (Boolean) Whether the In-App Purchase is automatically made available in territories Apple adds in the future. Changing this forces a new resource.
- `available_territories` (Set of String) The set of territory codes (e.g. `USA`, `FRA`) where the In-App Purchase is available. Changing the set forces a new resource.
- `in_app_purchase_id` (String) The ID of the In-App Purchase. Changing this forces a new resource.

### Read-Only

- `id` (String) The unique identifier of the availability configuration.

## Import

Availability configurations can be imported using their ID:

```bash
terraform import appleappstoreconnect_in_app_purchase_availability.premium XXXXXXXXXX
```

~> **Note** App Store Connect does not expose a delete endpoint for availability; it is replaced by applying a new one. Removing this resource only drops it from Terraform state.
