# appleappstoreconnect_in_app_purchase_price_schedule (Resource)

Manages the price schedule of an In-App Purchase. An In-App Purchase has a single price schedule; App Store Connect derives prices for other territories from the base territory, and any explicit per-territory prices are provided via `manual_prices`. Price point IDs are resolved with the `appleappstoreconnect_in_app_purchase_price_point` data source. Changing any attribute replaces the schedule.

## Example Usage

```hcl
resource "appleappstoreconnect_in_app_purchase" "premium" {
  app_id               = data.appleappstoreconnect_app.example.id
  product_id           = "com.example.app.premium"
  name                 = "Premium"
  in_app_purchase_type = "NON_CONSUMABLE"
}

data "appleappstoreconnect_in_app_purchase_price_point" "usd_099" {
  in_app_purchase_id = appleappstoreconnect_in_app_purchase.premium.id
  territory          = "USA"
  customer_price     = "0.99"
}

resource "appleappstoreconnect_in_app_purchase_price_schedule" "premium" {
  in_app_purchase_id = appleappstoreconnect_in_app_purchase.premium.id
  base_territory     = "USA"

  manual_prices = [
    {
      price_point_id = data.appleappstoreconnect_in_app_purchase_price_point.usd_099.id
      territory      = "USA"
    },
  ]
}
```

## Schema

### Required

- `base_territory` (String) The base territory code (e.g. `USA`) Apple uses to derive prices in other territories. Changing this forces a new resource.
- `in_app_purchase_id` (String) The ID of the In-App Purchase this schedule prices. Changing this forces a new resource.
- `manual_prices` (Attributes Set) The explicit per-territory prices. At minimum the base territory must be included. Changing the set forces a new resource. (see [below for nested schema](#nestedatt--manual_prices))

### Read-Only

- `id` (String) The unique identifier of the price schedule.

<a id="nestedatt--manual_prices"></a>
### Nested Schema for `manual_prices`

Required:

- `price_point_id` (String) The price point ID for this territory (resolve it with the `appleappstoreconnect_in_app_purchase_price_point` data source).
- `territory` (String) The territory code this price applies to (e.g. `USA`).

Optional:

- `start_date` (String) The date the price takes effect (ISO 8601, e.g. `2025-06-01`). Omit for immediate effect.

## Import

Price schedules can be imported using their ID:

```bash
terraform import appleappstoreconnect_in_app_purchase_price_schedule.premium XXXXXXXXXX
```

~> **Note** App Store Connect does not expose a delete endpoint for price schedules; a new schedule replaces the previous one. Removing this resource only drops it from Terraform state.
