# appleappstoreconnect_in_app_purchase (Resource)

Manages an In-App Purchase (consumable, non-consumable, or non-renewing subscription) for an app in App Store Connect. Auto-renewable subscriptions are managed with the `appleappstoreconnect_subscription` resource instead.

In-App Purchases are bound to a single app and cannot be shared between apps. To
manage the same product across several apps (e.g. production and staging), define
it once and instantiate it per app — see the `appleappstoreconnect_app` data
source and `for_each`.

## Example Usage

### Basic Example

```hcl
data "appleappstoreconnect_app" "example" {
  bundle_id = "com.example.app"
}

resource "appleappstoreconnect_in_app_purchase" "premium" {
  app_id               = data.appleappstoreconnect_app.example.id
  product_id           = "com.example.app.premium"
  name                 = "Premium"
  in_app_purchase_type = "NON_CONSUMABLE"
  family_sharable      = true
}
```

### Consumable with a review note

```hcl
resource "appleappstoreconnect_in_app_purchase" "coins" {
  app_id               = data.appleappstoreconnect_app.example.id
  product_id           = "com.example.app.coins_100"
  name                 = "100 Coins"
  in_app_purchase_type = "CONSUMABLE"
  review_note          = "Grants 100 in-game coins."
}
```

## Schema

### Required

- `app_id` (String) The App Store Connect ID of the app this In-App Purchase belongs to. Use the `appleappstoreconnect_app` data source to look it up. Changing this forces a new resource.
- `in_app_purchase_type` (String) The type of In-App Purchase. One of `CONSUMABLE`, `NON_CONSUMABLE`, or `NON_RENEWING_SUBSCRIPTION`. Changing this forces a new resource.
- `name` (String) The reference name of the In-App Purchase, used in App Store Connect and Sales and Trends reports (max 64 characters). This can be updated in place.
- `product_id` (String) The unique product ID of the In-App Purchase (e.g., `com.example.app.premium`). Must be unique within the app. Changing this forces a new resource.

### Optional

- `family_sharable` (Boolean) Whether the In-App Purchase is available through Family Sharing. Defaults to the value assigned by App Store Connect.
- `review_note` (String) A note for the App Review team (max 4000 characters).

### Read-Only

- `id` (String) The unique identifier of the In-App Purchase.
- `state` (String) The state of the In-App Purchase (e.g., `MISSING_METADATA`, `READY_TO_SUBMIT`, `APPROVED`).

## Import

In-App Purchases can be imported using their ID:

```bash
terraform import appleappstoreconnect_in_app_purchase.example XXXXXXXXXX
```

Where `XXXXXXXXXX` is the In-App Purchase ID from App Store Connect (not the product ID).
