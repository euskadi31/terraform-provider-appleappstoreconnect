# appleappstoreconnect_in_app_purchase_localization (Resource)

Manages a localized name and description for an In-App Purchase in App Store Connect. At least one localization (typically the app's primary locale) is required before an In-App Purchase can be submitted for review.

## Example Usage

### Single locale

```hcl
resource "appleappstoreconnect_in_app_purchase_localization" "en" {
  in_app_purchase_id = appleappstoreconnect_in_app_purchase.premium.id
  locale             = "en-US"
  name               = "Premium"
  description        = "Unlock all premium features."
}
```

### Multiple locales with for_each

```hcl
locals {
  localizations = {
    "en-US" = { name = "Premium", description = "Unlock all premium features." }
    "fr-FR" = { name = "Premium", description = "Débloquez toutes les fonctionnalités premium." }
  }
}

resource "appleappstoreconnect_in_app_purchase_localization" "this" {
  for_each = local.localizations

  in_app_purchase_id = appleappstoreconnect_in_app_purchase.premium.id
  locale             = each.key
  name               = each.value.name
  description        = each.value.description
}
```

## Schema

### Required

- `in_app_purchase_id` (String) The ID of the In-App Purchase this localization belongs to. Changing this forces a new resource.
- `locale` (String) The locale of the localization (e.g., `en-US`, `fr-FR`). Changing this forces a new resource.
- `name` (String) The localized display name of the In-App Purchase (max 30 characters).

### Optional

- `description` (String) The localized description of the In-App Purchase (max 4000 characters).

### Read-Only

- `id` (String) The unique identifier of the In-App Purchase localization.

## Import

In-App Purchase localizations can be imported using their ID:

```bash
terraform import appleappstoreconnect_in_app_purchase_localization.en XXXXXXXXXX
```

Where `XXXXXXXXXX` is the localization ID from App Store Connect.
